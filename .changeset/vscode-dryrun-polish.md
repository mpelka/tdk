Polish the Backstage dry-run integration in the TDK Trace panel — fidelity fixes, a run history, and a configuration-gated Dry-run button.

- **Skipped steps normalize as skipped.** A dry-run step whose `if:` condition was false now renders the ⤼ skipped glyph (mirroring the local trace), read from the structured `status: "skipped"` field on the scaffolder's own `Skipping step … because its if condition was false` log line (the field, never the spoofable message text — a step whose output echoes that sentence stays ✓), with a "skipped — its `if:` was false" note in the detail body. Before, it wrongly showed the ✓ ran glyph.
- **Expression-only inputs — never `key: undefined`.** A provenance leaf that carries a source expression but never resolved a value (a skipped step, or any step that never ran) now renders the `${{ … }}` expression alone, with no `→` arrow and no fabricated `undefined`. Before, such a leaf dropped the expression and printed `key: undefined`.
- **Log de-noising.** The per-step Log section now reads as a clean run narrative: the double-encoded "Running … with inputs (secrets redacted): { … }" dump (already shown properly in the Inputs section above) collapses to `… (inputs shown above)`, the noise `info:` level prefix is stripped, and `warn:` / `error:` lines are kept and marked so signal stands out.
- **Dry-run run history.** Each preview keeps a capped list (last 20) of completed dry-runs. The dry-run slot header gains a `Run N of M · <time>` indicator with ‹ › navigation (older/newer); navigating replays that run through the same per-source seq-guarded post path a fresh dry-run uses, so a stale in-flight dry-run can't clobber it. EVERY completed run is recorded — one finishing after a newer stamp (a rapid resubmit, or a navigation mid-flight) still lands in history; only its auto-show is suppressed, and the shown run's indicator count refreshes in place (no tab switch, no run replaced). History survives tab flips and preview focus switches (the focus-switch replay restores the selected run) and is cleared when the preview is disposed. Failed runs are history entries too, labeled by taxonomy.
- **Dry-run button gated on configuration.** When `tdk.backstage.baseUrl` is unset, the "Dry-run in Backstage" button is disabled with a visible hint naming the `TDK: Set Backstage Base URL` and `TDK: Set Backstage Token` commands. The extension posts the capability with the panel's initial state and on `workspace.onDidChangeConfiguration`, so setting the base URL live re-enables the button with no reload. A missing token does not disable the button (some backends accept anonymous requests; the authFailed taxonomy already guides).
- **Unmapped log lines are never dropped.** Log entries for a step id the adapter can't map to a template step continue to surface in the run-log/preamble section. (Backstage's task banner counts one implicit trailing step beyond the template's rail — e.g. "Starting up task with 4 steps" for a 3-step template — which emits no log line, so nothing is lost.)

The skipped-step detection, expression-only fallback, log de-noising, and run-history model are pure lib seams with unit tests (the adapter and de-noiser are pinned against a second captured real Backstage response with a skipped step, redacted to the bakery theme). The reworked slot header, gated button, and navigation are covered by React Testing Library tests and the built-bundle smoke test.
